# 2026-04-27

## Debugging: Neovim Tree-sitter Nix Parser Error

### Request
User reported a `noice.nvim` error when opening `~/.config/nix/mbp2023/flake.nix`:
```
...d-0.12.1/share/nvim/runtime/lua/vim/treesitter/query.lua:374: Query error at 74:3.
Invalid field name "operator": operator: _ @operator)
```

### Investigation

**Files examined:**
- `~/.local/share/nvim/lazy/nvim-treesitter/runtime/queries/nix/highlights.scm` — confirmed it references `operator: _ @operator` in `unary_expression` and `binary_expression` nodes (lines 178–183).
- `~/.local/share/nvim/lazy/nvim-treesitter/lua/nvim-treesitter/parsers.lua` — confirmed the expected grammar revision is `eabf96807ea4ab6d6c7f09b671a88cd483542840`.
- `~/.local/share/nvim/lazy/nvim-treesitter/parser-info/nix.revision` — installed parser revision was `cfc53fd287d23ab7281440a8526c73542984669b` (outdated).
- `~/.local/share/nvim/lazy/nvim-treesitter/parser/nix.so` — binary compiled 13 Mar 2025, predating the grammar change that added `operator` as a named field.

**Root cause:** `lazy.nvim` updated `nvim-treesitter` on 2026-04-26, pulling in new Nix queries that reference the `operator` named field. The compiled `nix.so` was not recompiled to match. `:TSUpdate nix` incorrectly reported "all parsers up-to-date" due to a stale revision check.

### Solution
`:TSInstall! nix` inside neovim — the `!` forces recompilation regardless of the up-to-date check — recompiled `nix.so` from the correct grammar revision and resolved the error.

**Fallback:**
```bash
rm ~/.local/share/nvim/lazy/nvim-treesitter/parser/nix.so
rm ~/.local/share/nvim/lazy/nvim-treesitter/parser-info/nix.revision
# then :TSInstall nix inside neovim
```
